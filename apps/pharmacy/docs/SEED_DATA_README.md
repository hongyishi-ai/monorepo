# 种子数据和迁移脚本说明

本文档说明了任务 2.6 "创建种子数据和迁移脚本" 的实现内容。

## 已完成的子任务

### ✅ 创建初始管理员用户

- 在 `seed-data.sql` 中定义了三个测试用户：
  - 管理员 (admin@pharmacy.com)
  - 经理 (manager@pharmacy.com)
  - 操作员 (operator@pharmacy.com)
- 在 `reset-dev-data.js` 中提供了程序化创建用户的功能

### ✅ 添加示例药品数据

- 创建了 20 种常用药品的完整数据集
- 包含药品的完整信息：条码、名称、规格、厂家、货架位置、安全库存
- 涵盖多个药品类别：抗生素、解热镇痛、慢性病用药、中成药、外用药
- 为每种药品自动生成 1-3 个不同批次的数据

### ✅ 创建系统默认配置

- 定义了 10 个系统配置项：
  - 近效期提醒天数 (30天)
  - 库存不足提醒阈值 (10)
  - 扫码超时时间 (30秒)
  - 扫码重试次数 (3次)
  - 用户会话超时 (1小时)
  - 数据备份频率 (24小时)
  - 最大上传文件大小 (10MB)
  - 声音通知开关
  - 默认货架位置
  - 先进先出开关

### ✅ 编写数据库迁移脚本

- `migration.sql`: 完整的数据库迁移脚本
  - 包含所有表结构、索引、触发器、函数
  - 支持事务处理和错误回滚
  - 包含迁移版本管理
- `setup-database.js`: 自动化数据库初始化脚本
  - 支持命令行参数控制
  - 包含错误处理和恢复机制
  - 提供详细的执行日志

### ✅ 创建开发环境数据重置脚本

- `reset-dev-data.js`: 开发环境数据重置脚本
  - 安全的数据清理（仅清理测试数据）
  - 重新插入完整的种子数据
  - 数据完整性验证
- `test-migration-setup.js`: 迁移和种子数据测试脚本
  - 全面的数据库状态检查
  - 数据关系验证
  - 视图和函数测试

## NPM 脚本

新增了以下 NPM 脚本：

```json
{
  "setup:database": "初始化数据库架构",
  "setup:database:with-seed": "初始化数据库并插入种子数据",
  "reset:dev-data": "重置开发环境数据",
  "db:migrate": "强制重新迁移数据库",
  "db:seed": "插入种子数据",
  "test:migration": "测试迁移和种子数据"
}
```

## 使用方法

### 首次设置

```bash
npm run setup:database:with-seed
```

### 开发中重置数据

```bash
npm run reset:dev-data
```

### 测试数据库状态

```bash
npm run test:migration
```

## 注意事项

1. **环境变量**: 需要正确配置 Supabase 连接信息
2. **权限**: 需要服务角色密钥进行管理操作
3. **数据安全**: 生产环境不要使用种子数据
4. **版本管理**: 迁移脚本支持版本控制和重复执行
