## 🚀 执行步骤

### 1. 数据库设置

请在Supabase SQL编辑器中执行 `JWT_SETUP_STEPS.md` 中的6个步骤：

1. ✅ 创建JWT元数据同步函数
2. ✅ 创建触发器
3. ✅ 创建优化的权限检查函数
4. ✅ 创建辅助函数
5. ✅ 同步现有用户元数据
6. ✅ 验证设置

### 2. 验证设置

```bash

```

### 3. 重启应用

```bash

npm run dev
```

## 🛠 技术实现

### 1. 数据库层面

- **触发器同步**: 当用户信息更新时，自动同步到JWT元数据
- **优化函数**: 权限检查优先使用JWT中的角色信息
- **回退机制**: JWT信息缺失时回退到数据库查询

### 2. 前端层面

- **优先JWT**: 登录时优先使用JWT中的用户信息
- **减少查询**: 避免不必要的数据库查询
- **缓存优化**: 利用JWT的自然缓存特性

### 3. 安全保障

- **数据一致性**: 触发器确保JWT与数据库同步
- **权限验证**: 前后端双重权限检查
- **优雅降级**: 确保系统在各种情况下都能正常工作

## 🔧 故障排除

### 常见问题

1. **SQL语法错误**
   - 使用 `jwt-metadata-sync-fixed.sql` 而不是原始文件
   - 按照 `JWT_SETUP_STEPS.md` 分步执行

2. **函数不存在**
   - 检查是否所有SQL步骤都已执行
   - 运行 `npm run jwt:verify` 验证

3. **JWT中没有角色信息**
   - 检查触发器是否正常工作
   - 重新执行步骤5同步用户元数据

4. **性能改进不明显**
   - 确认前端代码已更新
   - 清除浏览器缓存重新登录
