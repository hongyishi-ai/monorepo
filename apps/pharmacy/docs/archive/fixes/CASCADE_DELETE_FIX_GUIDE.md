# 级联删除问题修复指南

## 问题描述

在药品管理页面尝试删除药品或批次时，出现以下错误：

```
Key is still referenced from table "inventory_transactions"
update or delete on table "medicines" violates foreign key constraint "inventory_transactions_medicine_id_fkey" on table "inventory_transactions"
```

这是因为数据库中存在外键约束，当药品或批次在 `inventory_transactions` 表中有相关记录时，无法直接删除。

## 解决方案

实现级联删除功能，在删除药品或批次前，先删除相关的库存交易记录，并向用户显示警告信息。

## 修复内容

### 1. 数据库函数

创建了以下 PostgreSQL 函数：

- `cascade_delete_medicine(UUID)` - 级联删除药品
- `cascade_delete_batch(UUID)` - 级联删除批次
- `safe_delete_medicine(UUID, BOOLEAN)` - 安全删除药品（带确认）
- `safe_delete_batch(UUID, BOOLEAN)` - 安全删除批次（带确认）
- `get_medicine_dependencies(UUID)` - 检查药品依赖关系
- `get_batch_dependencies(UUID)` - 检查批次依赖关系

### 2. 前端组件

- 新增 `CascadeDeleteDialog` 组件，显示删除警告和依赖信息
- 更新 `MedicineManagementPage` 组件，支持级联删除流程
- 新增 `use-cascade-delete.ts` hooks，处理级联删除逻辑

### 3. 服务层更新

- 更新 `medicineService` 和 `batchService`，添加级联删除方法
- 改进错误处理，提供更友好的错误信息

## 部署步骤

### 步骤 1：部署数据库函数

1. 运行部署脚本查看 SQL 内容：

   ```bash
   node scripts/deploy-cascade-delete.js
   ```

2. 打开 Supabase 控制台 → SQL 编辑器

3. 复制并执行 `supabase/cascade-delete-functions.sql` 文件中的所有 SQL 语句

### 步骤 2：测试函数部署

运行测试脚本验证函数是否正确部署：

```bash
node scripts/test-cascade-delete.js
```

### 步骤 3：重启应用

重启前端应用以加载新的代码：

```bash
npm run dev
```

## 功能说明

### 删除流程

1. **预检查**：用户点击删除时，系统首先检查是否有依赖数据
2. **警告提示**：如果有依赖数据，显示级联删除警告对话框
3. **用户确认**：用户确认后执行级联删除
4. **级联删除**：自动删除相关的库存交易记录，然后删除目标项

### 用户界面

- **简单删除**：无依赖数据时，显示普通删除确认对话框
- **级联删除**：有依赖数据时，显示详细的警告信息，包括：
  - 将要删除的交易记录数量
  - 将要删除的批次数量（仅药品删除）
  - 明确的警告文字和图标

### 安全特性

- **二次确认**：所有删除操作都需要用户确认
- **详细信息**：清楚显示将要删除的关联数据
- **事务安全**：使用数据库事务确保数据一致性
- **权限检查**：保持原有的权限验证机制

## 测试验证

### 测试场景

1. **删除无依赖的药品/批次**：应该正常删除
2. **删除有依赖的药品/批次**：应该显示警告并支持级联删除
3. **取消删除操作**：应该正确取消，不删除任何数据
4. **权限验证**：无权限用户应该无法删除

### 验证步骤

1. 创建测试药品和批次
2. 进行入库操作，创建库存交易记录
3. 尝试删除批次，验证警告提示
4. 确认删除，验证级联删除效果
5. 尝试删除药品，验证完整的级联删除

## 错误处理

### 常见错误

1. **函数不存在**：
   - 检查是否正确执行了 SQL 函数
   - 验证 Supabase 连接和权限

2. **权限不足**：
   - 确保用户有删除权限
   - 检查 RLS 策略设置

3. **数据完整性错误**：
   - 检查外键约束设置
   - 验证数据关联关系

### 错误信息

系统会显示友好的错误信息：

- "该药品存在关联的库存或批次记录，无法删除"
- "您没有权限删除该药品"
- "删除操作失败，请稍后重试"
