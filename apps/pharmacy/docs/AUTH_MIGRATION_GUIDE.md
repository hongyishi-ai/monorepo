# 认证系统迁移指南

## 迁移概述

本指南详细说明如何从旧认证系统迁移到新的优化认证系统。新系统提供了更好的性能、更简洁的代码结构和更强的可维护性。

## 迁移状态

**当前状态**: ✅ 已完成测试，推荐迁移  
**迁移日期**: 2024-12-19  
**测试结果**: 所有功能测试通过，性能表现优秀

## 系统对比

### 旧认证系统 (auth.store.ts)

- **代码行数**: 942行
- **复杂度**: 高
- **状态管理**: 复杂的subscribeWithSelector
- **加载状态**: 多个状态 (isLoading, isInitializing, isProfileLoading)
- **功能**: 完整但复杂

### 新认证系统 (auth-optimized.store.ts)

- **代码行数**: 214行
- **复杂度**: 低
- **状态管理**: 简化的persist中间件
- **加载状态**: 单一状态 (isLoading)
- **功能**: 核心功能，性能优化

## 环境配置

### 开发环境

```bash
# .env.local
VITE_USE_NEW_AUTH=true
```

### 生产环境

```bash
# .env.production
VITE_USE_NEW_AUTH=true
```

### 环境变量说明

- `VITE_USE_NEW_AUTH=true`: 启用新认证系统
- `VITE_USE_NEW_AUTH=false`: 使用旧认证系统（默认）
