# 药房业务规则

## 药品管理规则

### 药品信息规范
- 药品条码必须唯一，不允许重复
- 药品名称、规格、生产厂家为必填字段
- 货架位置格式：区域-货架-层级 (如 A-01-3)
- 安全库存不能为负数，默认值为 0

### 批次管理规则
- 每个药品可以有多个批次
- 批次号在同一药品下必须唯一
- 生产日期不能晚于当前日期
- 有效期必须晚于生产日期
- 批次数量不能为负数

## 库存操作规则

### 入库规则
- 扫码入库时，如果是新药品，必须填写完整的药品信息
- 入库数量必须大于 0
- 批次信息必须完整填写
- 首次入库的药品必须设置货架位置
- 相同批次的药品入库会累加数量

### 出库规则
- 出库数量不能超过当前库存
- 自动按照先进先出 (FIFO) 原则选择批次
- 优先使用临近有效期的批次
- 出库后如果批次数量为 0，标记批次为已用完
- 出库操作必须记录操作人员信息

### 库存计算规则
- 当前库存 = 所有有效批次的数量总和
- 有效批次：未过期且数量大于 0 的批次
- 库存更新必须实时反映在系统中
- 支持库存盘点和调整功能

## 提醒和预警规则

### 近效期提醒
- 默认提醒阈值：30 天
- 管理员可以自定义提醒天数
- 近效期药品按剩余天数升序排列
- 已过期药品单独标记和处理

### 库存不足提醒
- 当库存数量低于安全库存时触发提醒
- 安全库存可以针对每种药品单独设置
- 提醒信息包含建议采购数量
- 库存恢复后自动取消提醒

## 用户权限规则

### 角色定义
- **管理员 (admin)**: 所有权限，包括用户管理、系统配置
- **经理 (manager)**: 库存管理、报表查看、数据导入导出
- **操作员 (operator)**: 基本的入库出库操作

### 权限矩阵
- 药品信息管理：管理员、经理
- 入库操作：所有角色
- 出库操作：所有角色
- 库存查询：所有角色
- 报表查看：管理员、经理
- 数据导入导出：管理员、经理
- 用户管理：仅管理员
- 系统配置：仅管理员

## 数据完整性规则

### 必填字段验证
- 药品：条码、名称、规格、生产厂家
- 批次：批次号、生产日期、有效期、数量
- 库存交易：药品ID、批次ID、操作类型、数量

### 数据关联规则
- 删除药品前必须确保没有关联的库存记录
- 删除批次前必须确保库存数量为 0
- 用户删除前必须转移其操作记录的归属

### 审计日志规则
- 所有库存变动必须记录操作日志
- 日志包含：操作时间、操作人员、操作类型、变动数量
- 日志数据不允许修改或删除
- 支持按时间范围和操作类型查询日志

## 扫码操作规则

### 条码格式支持
- 支持 EAN-13、Code 128、QR Code 等常见格式
- 条码长度限制：6-20 位字符
- 支持数字和字母组合的条码

### 扫码错误处理
- 无效条码：显示"条码格式错误"提示
- 未找到药品：提示是否添加新药品
- 扫码超时：自动重新激活扫码功能
- 摄像头权限被拒绝：显示权限申请指导

## 数据导入导出规则

### 导入规则
- 支持 Excel (.xlsx) 和 CSV 格式
- 必须包含模板中的必填列
- 数据验证失败时显示详细错误信息
- 支持部分成功导入，跳过错误行

### 导出规则
- 支持按时间范围导出
- 支持按药品类别导出
- 导出文件包含完整的字段信息
- 敏感信息需要权限控制

## 系统配置规则

### 可配置项
- 近效期提醒天数
- 系统超时时间
- 数据备份频率
- 用户会话有效期

### 配置变更规则
- 配置变更需要管理员权限
- 重要配置变更需要记录日志
- 配置变更后需要通知相关用户
