# FMS 应用测试文档

本文档详细说明了FMS（功能性动作筛查）应用的测试套件，包括如何运行测试、理解测试结果，以及测试覆盖的功能范围。

## 🚀 快速开始

### 运行所有测试
```bash
npm test
```

### 运行特定测试套件
```bash
# 运行应用主测试
npm test App.test.tsx

# 运行评估功能测试
npm test Assessment.test.tsx

# 运行报告生成测试
npm test Report.test.tsx

# 运行集成测试
npm test Integration.test.tsx

# 运行组件测试
npm test Components.test.tsx
```

### 生成测试覆盖率报告
```bash
npm run test:coverage
```

### 使用可视化界面运行测试
```bash
npm run test:ui
```

## 🚨 测试场景

### 正常流程测试
1. **完美评估**: 所有测试得满分，获得A级评价
2. **普通评估**: 混合分数，获得B或C级评价
3. **需要改善**: 低分结果，获得D级评价

### 异常情况测试
1. **疼痛场景**: 测试中出现疼痛（0分），显示安全警告
2. **排除测试失败**: 排除性测试未通过，显示医疗建议
3. **数据缺失**: 处理不完整的评估数据

### 边界条件测试
1. **快速点击**: 防止重复提交和状态混乱
2. **内存泄漏**: 组件正确卸载和清理
3. **无效路由**: 错误URL的处理

## ⚠️ 已知问题和改进建议

### 高优先级问题 🔴

1. **可访问性改进**
   - 问题: 某些动态内容缺少屏幕阅读器支持
   - 建议: 添加aria-live和更详细的aria标签
   - 影响: 视觉障碍用户体验

2. **数据验证加强**
   - 问题: 输入验证可能不够严格
   - 建议: 增强客户端和服务端验证规则
   - 影响: 数据安全性和可靠性

### 中优先级问题 🟡

3. **性能优化**
   - 问题: 大量数据渲染时可能出现性能问题
   - 建议: 考虑实现虚拟滚动或分页加载
   - 影响: 用户体验流畅度

4. **移动端体验**
   - 问题: 移动端触摸交互可能不够流畅
   - 建议: 优化触摸事件处理和手势支持
   - 影响: 移动端用户体验

## 🛠️ 测试最佳实践

### 开发流程
1. **开发前**: 编写失败的测试用例（TDD）
2. **开发中**: 保持测试通过，及时修复失败测试
3. **开发后**: 运行完整测试套件，确保无回归

### 测试维护
1. **定期更新**: 随功能更新测试用例
2. **重构清理**: 删除过时的测试，优化测试性能
3. **文档同步**: 保持测试文档与代码同步

### 调试技巧
1. **单独运行**: 隔离问题测试用例
2. **详细日志**: 使用console.log调试测试状态
3. **快照测试**: 对比UI变化

## 🔧 故障排除

### 常见问题

**问题**: 测试运行缓慢
```bash
# 解决方案: 并行运行测试
npm test -- --reporter=verbose --pool=threads
```

**问题**: 模拟组件报错
```bash
# 解决方案: 检查mock配置
vi.mock('framer-motion', ...)
```

**问题**: 异步测试失败
```bash
# 解决方案: 增加等待时间
await waitFor(() => {...}, { timeout: 5000 })
```
