# 功能性动作筛查(FMS)交互式项目PRD文档

## 一、项目概述

### 1.1 项目目标

开发一款基于FMS的纯前端交互式运动风险评估与针对性强化训练应用，帮助用户识别运动风险、改善动作质量、预防损伤，同时确保完全在客户端运行，不收集用户数据。

### 1.2 用户问题

用户缺乏便捷、科学且隐私安全的方法来评估自身运动功能风险，难以获得个性化的纠正性训练指导。

### 1.3 产品定位

一款专业的、个性化的、纯前端交互式运动健康辅助工具，无需账户注册，无数据追踪，用完即走。

### 1.4 核心价值

* 早期风险识别

* 科学训练指导

* 提升运动表现

* 降低损伤概率

* 保护用户隐私

### 1.5 技术约束

* 纯前端实现，可在Cloudflare上部署运行

* 不依赖后端服务器

* 不收集、存储或传输用户数据

* 所有计算和处理在客户端完成

## 二、用户画像

### 2.1 健身爱好者/运动员

* **基本信息**：18-45岁，有规律运动习惯

* **痛点**：希望优化动作、提升表现、预防运动损伤

* **使用场景**：训练前自我评估，发现潜在风险点

* **隐私需求**：不希望个人运动数据被收集分析

### 2.2 普通运动人群

* **基本信息**：25-60岁，偶尔参与体育活动

* **痛点**：担心运动损伤，希望了解自身功能限制

* **使用场景**：开始新运动前进行风险评估

* **隐私需求**：希望匿名使用，不留下数字足迹

### 2.3 康复后期人群

* **基本信息**：任何年龄段，曾经受伤正在恢复

* **痛点**：需要安全有效地恢复功能性动作能力

* **使用场景**：在专业指导下，通过功能性训练恢复运动能力

* **隐私需求**：健康数据敏感，拒绝数据追踪

## 三、功能规格

### 3.1 无账户访问模块

* **3.1.1 匿名访问**

  * **需求描述**：用户无需注册账户或提供任何个人信息即可使用全部功能

  * **验收标准**：应用无任何注册/登录入口；无需Cookie或本地存储即可完整使用

* **3.1.2 本地数据处理**

  * **需求描述**：所有用户输入的数据仅在浏览器内存中处理，不发送至任何服务器

  * **验收标准**：网络监控工具显示无数据传出；刷新页面后所有数据重置

* **3.1.3 可选本地存储**

  * **需求描述**：提供选项让用户自行决定是否在本地设备临时保存结果（如PDF导出）

  * **验收标准**：明确告知用户数据仅保存在其设备上；提供清晰的导出选项

### 3.2 FMS评估模块

* **3.2.1 测试引导**

  * **需求描述**：系统需为7个FMS测试  及3个排除性测试  提供清晰的视频、图片和文字指导

  * **验收标准**：视频加载流畅；图文说明清晰；所有资源预加载至客户端

* **3.2.2 数据输入与评分**

  * **需求描述**：支持用户自评打分(0-3分) ；记录疼痛与否

  * **评分逻辑**：严格按照FMS评分标准 。若排除性测试阳性（疼痛），对应主测试记0分

  * **验收标准**：评分界面直观；疼痛记录选项明确；所有计算在客户端JavaScript中完成

### 3.3 风险分析与报告模块

* **3.3.1 FMS报告生成**

  * **需求描述**：自动计算总分、各单项得分；识别并展示不对称性；列出与得分相关的潜在功能限制和常见代偿模式

  * **生物力学解读**：结合相关肌肉功能（主动肌、协同肌、稳定肌）和关节运动模式解释限制原因

  * **验收标准**：报告内容准确、全面；图表清晰易懂；支持本地PDF导出

* **3.3.2 疼痛与0分处理**

  * **需求描述**：若出现0分或排除性测试阳性，系统需显著提示风险，解释可能的临床意义（如肩峰下撞击、椎间盘问题等 ），并强烈建议用户寻求专业医疗咨询

  * **验收标准**：风险提示醒目；临床意义解释准确；转介建议明确

### 3.4 个性化训练模块

* **3.4.1 训练计划推荐逻辑**

  * **需求描述**：根据FMS报告（最低分、不对称性、功能限制），推荐针对性的纠正/强化练习。练习选择基于改善活动度、柔韧性、稳定性、神经肌肉控制和对称性的原则

  * **优先级**：优先处理1分项目，其次是2分项目和不对称性。0分项目（疼痛）需在医疗专业人士评估并许可后方可介入

  * **验收标准**：推荐算法完全在客户端JavaScript中实现；训练计划与FMS结果高度相关且科学合理

* **3.4.2 动作库**

  * **需求描述**：包含每个练习的视频演示、文字描述、目标肌肉图示 、常见错误提示

  * **验收标准**：视频预加载至客户端；描述准确；覆盖FMS各方面需求的练习

* **3.4.3 训练执行指导**

  * **需求描述**：提供动作执行的详细指导，包括组数、次数建议和注意事项

  * **验收标准**：指导内容科学合理；界面直观易用

### 3.5 教育内容模块

* **3.5.1 FMS知识库**

  * **需求描述**：提供FMS基础知识、评分标准解释、生物力学原理等静态内容

  * **验收标准**：内容科学准确；加载迅速；无需联网即可访问所有内容

* **3.5.2 常见问题解答**

  * **需求描述**：针对FMS测试和训练的常见问题提供解答

  * **验收标准**：问答内容全面；界面易于导航

## 四、非功能性需求

### 4.1 性能要求

* **4.1.1 加载速度**

  * 首次加载时间<3秒（在4G网络环境下）

  * 所有资源（视频、图片）需预加载或实现渐进式加载

  * 应用大小优化，总体积<5MB（不含视频资源）

* **4.1.2 运行性能**

  * 在中端移动设备上流畅运行，无卡顿

  * 动画和交互响应时间<100ms

  * 计算密集型操作（如生成报告）<1秒完成

### 4.2 隐私保护

* **4.2.1 零数据收集**

  * 不使用任何分析工具或追踪脚本

  * 不使用第三方Cookie

  * 不收集使用数据或崩溃报告

* **4.2.2 透明度**

  * 在应用首页明确说明隐私政策：无数据收集、无追踪

  * 提供源代码查看链接，证明无数据传输

### 4.3 可用性

* **4.3.1 界面设计**

  * 符合直觉的操作流程

  * 清晰的视觉层次和信息架构

  * 适合各年龄段用户的易用性

* **4.3.2 无障碍性**

  * 符合WCAG 2.1 AA级标准

  * 支持屏幕阅读器

  * 提供足够的色彩对比度

### 4.4 兼容性

* **4.4.1 设备兼容**

  * 响应式设计，适配手机、平板和桌面设备

  * 支持横屏和竖屏模式

* **4.4.2 浏览器兼容**

  * 支持最新版Chrome、Safari、Firefox、Edge

  * 优雅降级，确保基本功能在旧版浏览器上可用

### 4.5 离线功能

* **4.5.1 PWA支持**

  * 实现Progressive Web App功能

  * 支持离线访问核心功能

  * 提供"添加到主屏幕"选项

## 五、技术架构

### 5.1 前端技术栈

* **5.1.1 核心框架**

  * 框架：React 18
  * 状态管理：React Context + 自定义Hooks

* **5.1.2 UI组件**

  * UI库：shadcn-ui (提供优秀的设计基础和可定制性)
  * 样式方案：Tailwind CSS (实现原子化CSS和响应式设计)
  * 动画：Framer Motion (实现流畅的交互动画)

* **5.1.3 资源优化**

  * 图片：WebP格式，适当压缩

  * 视频：使用HLS/DASH自适应码率

  * 代码分割和懒加载

### 5.2 数据处理

* **5.2.1 客户端计算**

  * 所有FMS评分计算在客户端JavaScript中实现

  * 训练推荐算法使用决策树或简单规则引擎

* **5.2.2 临时存储**

  * 使用浏览器内存（变量）存储会话数据

  * 可选的sessionStorage用于防止意外刷新丢失数据

  * 明确提示用户数据仅临时保存

### 5.3 部署架构

* **5.3.1 Cloudflare部署**

  * 利用Cloudflare Pages静态网站托管

  * 配置适当的缓存策略

  * 使用Cloudflare Workers实现必要的客户端API（如PDF生成）

* **5.3.2 资源分发**

  * 利用Cloudflare CDN加速全球访问

  * 实现资源预加载策略

  * 配置适当的缓存控制头

## 六、用户流程

### 6.1 首次访问流程

1. 用户访问应用首页

2. 显示简短介绍和隐私声明（强调无数据收集）

3. 提供两个选项：

   * 开始FMS评估

   * 浏览教育内容

### 6.2 FMS评估流程

1. 显示FMS测试概述和准备事项

2. 按顺序引导用户完成7项基础测试和3项排除性测试

3. 每项测试提供：

   * 视频示范

   * 文字说明

   * 评分标准

   * 自评输入界面

4. 完成所有测试后生成综合报告

### 6.3 报告查看流程

1. 显示FMS总分和各项得分

2. 提供图形化表示（如雷达图）

3. 列出识别的功能限制和代偿模式

4. 对0分项目给予特别警示

5. 提供导出PDF选项

### 6.4 训练推荐流程

1. 基于FMS结果显示个性化训练建议

2. 按优先级排序显示针对性练习

3. 每个练习提供：

   * 视频示范

   * 执行要点

   * 目标肌肉说明

   * 组数/次数建议

4. 允许用户选择感兴趣的练习查看详情

## 十、风险与缓解策略

### 10.1 已识别风险

* **纯前端限制**：复杂计算可能影响性能

  * **缓解**：优化算法，使用Web Workers分担计算负载

* **资源体积**：视频资源可能导致应用过大

  * **缓解**：实现渐进式加载，提供低/高质量选项

* **专业准确性**：确保内容专业性和科学性

  * **缓解**：邀请专业人士审核内容

* **用户误解**：用户可能误解自我评估结果

  * **缓解**：明确声明应用局限性，强调专业咨询的重要性

### 10.2 合规考虑

* **健康建议免责声明**：明确说明应用不提供医疗诊断

* **无障碍合规**：确保符合WCAG标准

* **隐私政策**：虽无数据收集，仍需提供简明隐私声明
