# FMS项目文档更新指南

## 📋 概述

本指南定义了FMS项目中文档与代码同步的规范和流程，确保文档始终反映项目的最新状态。

## 🔄 文档同步原则

### 1. 自动化优先
- 使用 `doc-sync-check.js` 脚本自动检查文档状态
- 测试结果自动更新到 `TEST_RESULTS.md`
- Git hooks 集成预提交检查

### 2. 及时更新
- 功能实现后立即更新相关文档
- 新增组件或页面时同步更新架构文档
- API变更时立即更新接口文档

### 3. 内容准确性
- 将已实现功能从"未来计划"移至"已实现功能"
- 移除过时的技术栈信息
- 更新版本号和依赖信息

## 📚 文档类型与更新触发条件

### 主要README文档

#### 根目录 README.md
**更新触发条件:**
- 新增页面或主要功能模块
- 技术栈变更
- 项目架构调整
- 部署方式改变

**需要更新的章节:**
- 功能特性列表
- 技术栈说明
- 快速开始指南
- 已实现的高级功能

#### fms-app/README.md
**更新触发条件:**
- 组件接口变更
- 状态管理模式调整
- 构建配置修改
- 开发工具变更

**需要更新的章节:**
- 项目结构说明
- 组件架构图
- 状态管理流程
- 开发指南

### 技术文档

#### DESIGN_SYSTEM.md
**更新触发条件:**
- UI组件新增或修改
- 设计规范变更
- 主题配置调整

#### TESTING.md
**更新触发条件:**
- 测试策略调整
- 新增测试套件
- 测试工具变更

#### TEST_RESULTS.md (自动更新)
**更新触发条件:**
- 每次运行 `npm run doc:sync` 时自动更新
- 测试通过率变化
- 新增或删除测试用例

## 🔧 自动化工具使用

### 文档同步检查脚本

```bash
# 运行完整的文档同步检查
npm run doc:sync

# 仅检查文件变更
npm run doc:check

# 更新测试状态
npm run doc:test-update
```

### 检查项目

1. **文件变更检查**: 检测代码文件变更，提醒相关文档更新
2. **功能状态同步**: 验证已实现功能是否正确反映在文档中
3. **测试状态更新**: 自动更新测试结果和覆盖率信息
4. **依赖版本检查**: 检查package.json变更是否需要更新文档

## 📝 文档更新工作流

### 开发阶段
1. **功能开发完成**
   - 运行 `npm run doc:sync` 检查
   - 根据提示更新相关文档
   - 提交代码前再次运行检查

2. **组件修改**
   - 更新组件文档和使用示例
   - 如有API变更，更新接口说明
   - 更新设计系统文档

3. **新功能实现**
   - 从"未来计划"移至"已实现功能"
   - 添加功能描述和使用指南
   - 更新项目架构图

### 发布阶段
1. **预发布检查**
   ```bash
   npm run doc:sync
   npm run test:run
   npm run build
   ```

2. **文档最终确认**
   - 验证所有功能状态正确
   - 检查版本号一致性
   - 确认部署说明准确

## 🎯 具体更新场景

### 场景1: 新增页面组件
**需要更新:**
- `README.md` - 功能模块列表
- `fms-app/README.md` - 项目结构
- 路由配置文档

**更新步骤:**
1. 在相应文档中添加页面描述
2. 更新项目架构图
3. 添加使用示例

### 场景2: 实现计划中的功能
**需要更新:**
- `README.md` - 从"未来计划"移至"已实现"
- `fms-app/README.md` - 功能详细说明

**更新步骤:**
1. 在"已实现的核心功能"中添加功能描述
2. 从"未来增强计划"中移除该功能
3. 添加使用说明和技术细节

### 场景3: 修改组件API
**需要更新:**
- `DESIGN_SYSTEM.md` - 组件接口说明
- `fms-app/README.md` - 组件架构

**更新步骤:**
1. 更新组件prop接口文档
2. 更新使用示例
3. 如有breaking change，添加迁移指南

### 场景4: 依赖升级
**需要更新:**
- `README.md` - 技术栈版本
- `fms-app/README.md` - 开发环境要求

**更新步骤:**
1. 更新依赖版本信息
2. 检查是否需要更新Node.js版本要求
3. 更新安装和构建说明

## 🚨 注意事项

### 必须更新的情况
- ❗ 新增或删除主要功能
- ❗ API接口变更
- ❗ 构建或部署流程变更
- ❗ 依赖重大升级

### 可选更新的情况
- 🔧 内部代码重构（不影响接口）
- 🔧 性能优化（不改变功能）
- 🔧 代码注释改进

### 文档质量检查清单
- [ ] 功能描述准确完整
- [ ] 示例代码可以运行
- [ ] 链接地址有效
- [ ] 版本信息正确
- [ ] 语言表达清晰
- [ ] 格式规范统一
