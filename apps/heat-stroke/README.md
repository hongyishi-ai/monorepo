# 热指数查询应用

这是一个简单的网页应用，用于查询并显示当前位置或指定城市的天气信息和热指数。

## 功能

*   **自动定位**：应用启动时会尝试获取用户的当前地理位置，并显示该位置的天气和热指数。
*   **手动搜索**：用户可以在输入框中输入城市名称，然后点击搜索按钮或按回车键来查询指定城市的天气信息。
*   **刷新数据**：用户可以点击刷新按钮重新获取当前位置的天气数据。
*   **当前天气显示**：
    *   天气图标和描述
    *   当前温度
    *   当前湿度
    *   当前风速
    *   计算出的当前热指数
*   **热指数警告**：根据当前热指数的级别显示不同颜色和内容的警告信息。
*   **24小时预报图表**：
    *   以折线图形式展示未来24小时（每3小时一个数据点）的温度、热指数和湿度变化趋势。
    *   图表背景根据热指数级别进行颜色分区显示（舒适、注意、警惕、危险、极度危险）。
    *   图表样式已适配深色主题，关键数据（热指数）使用亮橙色高亮。
*   **健康提示**：根据当前热指数级别提供相应的健康建议。
*   **滚动动画**：页面元素在滚动进入视口时具有淡入动画效果。

## 技术栈

*   **HTML**: 页面结构
*   **Tailwind CSS**: UI 样式 (通过 CDN 引入)
*   **JavaScript**: 交互逻辑、API 调用、热指数计算、图表生成
*   **OpenWeatherMap API**: 获取天气数据
*   **Chart.js**: 绘制天气预报图表 (通过 CDN 引入)
*   **chartjs-plugin-annotation**: Chart.js 插件，用于在图表上绘制背景区域 (通过 CDN 引入)
*   **Framer Motion**: 实现滚动动画效果 (通过 CDN 引入)
*   **Font Awesome**: 图标 (通过 CDN 引入)

## 如何运行

1.  确保你的电脑安装了 Python 3。
2.  在项目根目录（包含 `index.html` 文件的目录）下打开终端或命令行。
3.  运行一个简单的 HTTP 服务器。例如，使用 Python 的内置服务器：
    ```bash
    python3 -m http.server 8000
    ```
4.  在浏览器中打开 `http://localhost:8000`。

**注意**: 应用需要获取你的地理位置权限才能自动显示当前位置的天气。如果浏览器弹出请求权限的提示，请选择允许。

## 如何安装PWA应用

为了获得更好的应用体验，你可以将本网站作为PWA（Progressive Web App）安装到你的设备上：

### iOS设备（iPhone/iPad）
1. 使用Safari浏览器打开网站
2. 点击底部的"分享"按钮（方框加箭头图标）
3. 在弹出菜单中选择"添加到主屏幕"
4. 点击"添加"完成安装

### Android设备
1. 使用Chrome浏览器打开网站
2. 点击菜单按钮（右上角三个点）
3. 选择"安装应用"或"添加到主屏幕"
4. 按照提示完成安装

安装后，应用将以全屏模式运行，不会显示浏览器地址栏和导航栏，提供更原生的应用体验。

## 最近开发总结

### 1. PWA功能实现

* **问题**: 应用以PWA形式安装到手机上时，顶部会显示浏览器标题栏和地址栏，影响用户体验。
* **解决方案**:
  * 添加了Web App Manifest配置，设置display为standalone模式
  * 添加了iOS设备所需的apple-mobile-web-app-capable等meta标签
  * 配置了apple-mobile-web-app-status-bar-style为black-translucent，进一步优化顶部状态栏显示
  * 实现了Service Worker，支持应用资源缓存和离线访问
  * 创建了不同尺寸的应用图标，支持各种设备的显示需求
* **改进**:
  * 优化视口设置，禁止用户缩放以提供更原生的应用体验
  * 在README中添加了PWA安装指南，帮助用户更好地使用应用
  * 确保页面布局在standalone模式下正常显示

### 2. 库加载和显示问题修复 (之前的更新)

* **问题**: 页面打开后无法正常显示Chart.js图表和其他动态内容，只能看到标题和静态文本。
* **解决方案**:
  * 调整了JavaScript库的加载顺序，确保Chart.js在Framer Motion之前加载
  * 更新了库的URL路径，使用更可靠的CDN链接和完整路径
  * 添加了Chart.js和Framer Motion库加载失败的检测和错误处理
  * 增强了CSS样式，确保图表容器正确显示
  * 添加了降级显示机制，即使JavaScript库加载失败，页面也能显示基础内容
  * 增加了5秒后自动检测和恢复机制，如果页面未能加载数据，将自动搜索北京的天气
* **改进**:
  * 优化了Chart.js图表的样式和响应式显示
  * 增强了Framer Motion动画效果，确保在不同浏览器中的一致性
  * 添加了无障碍支持，当用户选择减少动画时，页面内容仍可正常显示

### 3. 滚动动画效果增强

*   实现了类似Apple官网的滚动动画效果，元素在进入视口时会有淡入和上移的动画。
*   使用Intersection Observer API监测元素进入视口的时机，自动触发动画。
*   添加了CSS过渡动画，实现基础的淡入和位移效果。
*   集成Framer Motion库，实现更复杂的动画序列和交错效果。
*   为不同元素设置了不同的动画延迟，创造错落有致的视觉效果。
*   优化了资源链接的悬停效果，增加了微妙的位移和阴影变化。

### 4. 项目结构优化

*   重新组织了项目文件结构，创建了assets和pages目录。
*   将CSS、JavaScript和HTML文件分类存放，便于日后扩展。
*   更新了所有文件的引用路径，确保功能正常。
*   添加了导航链接，方便在不同页面间切换。

### 9. 去重与稳态修复

*   **问题**：热指数查询页存在内联 CSS/JS 与 assets 版本重复，API 失败时用户只看到加载态且缺少兜底。
*   **解决方案**：
    *   移除页面内联样式与脚本，统一引用 `assets/css/styles.css` 与 `assets/js/script.js`，避免多处维护。
    *   为天气/地理接口增加超时与状态校验，定位失败自动切换到默认城市（北京）并提示用户。
    *   合并滚动动画观察者并托管返回顶部按钮，减少重复监听与潜在性能开销。

### 10. API Key 与本地缓存

*   **问题**：API Key 硬编码且重复请求影响响应速度。
*   **解决方案**：
    *   支持通过 `window.HS_API_KEY`、`meta[name="hs-api-key"]` 或 `localStorage.hs_api_key` 注入密钥，仍保留演示用默认值。
    *   为地理反查、城市搜索、当前天气、小时预报增加 10 分钟本地缓存；请求失败时可优先回退缓存数据。
    *   统一 Chart 注解插件注册与容器检查，避免空节点时报错。

### 11. 现场处置动效与交互稳态

*   **问题**：现场处置流程重复注册监听器、滚动动效单一。
*   **解决方案**：
    *   IntersectionObserver 结合 Framer Motion 实现淡入缩放，触发后解绑，减少开销。
    *   步骤 5/6 完成度判定改为单点监听 + 显示态过滤，避免多次进入步骤导致监听器累积。

### 12. 部署到 Vercel + API 代理

*   **部署平台**: Vercel
*   `api/openweather.js` 作为 Vercel Serverless Function：限制允许的 OpenWeatherMap 路径，并在服务端追加 `OPENWEATHER_API_KEY`。
*   环境变量：在 Vercel 项目设置中添加 `OPENWEATHER_API_KEY`。
*   前端配置：天气页已添加 `<meta name="hs-api-base" content="/api/openweather">`，`assets/js/script.js` 会优先走代理；若未检测到代理则回退直连。
