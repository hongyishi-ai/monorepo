# 技术文档

## "红医师"基层辅助用药系统

### 项目概述

本项目的目标是开发一款运行于安卓系统的应用程序，用于辅助基层门诊医生和卫生员在面对常见临床症状时，合理选择药品和处理方式。该程序是一个单页面应用程序（SPA），完全在前端运行，可离线安装。

### 主要开发步骤

1. **症状选择模块的实现**
- **目标**: 提供一个可交互的界面，允许用户同时选择多个症状及其子症状。
2. **推荐药物的即时反馈功能**
- **目标**: 根据用户选择的症状、剂型和成分，实时显示推荐药物。
3. **药物详细信息展示模块**
- **目标**: 当用户点击推荐药物时，显示药物的详细信息（如用法、剂量、禁忌症、药物相互作用等）。
4. **高级组合推荐逻辑**
- **目标**: 提供药物组合推荐，当没有单一药物符合所有条件时，给出多种药物的组合建议。
5. **动态数据加载和状态管理**
- **目标**: 动态获取药物、症状和诊断数据，确保应用数据的实时性和准确性。
6. **用户偏好设置模块的实现**
- **目标**: 向用户提供西医与祖国医学，是否含糖，性别，年龄等偏好设置。
7. **药物信息快速查询模块的实现**
- **目标**: 提供药物信息快速搜索和按照疾病、类别、剂型分类整理。提供药库数据批量替换。
8. **临床诊断思维链模块的实现**
- **目标**: 提供基层常见疾病的临床思维模型，确保医疗人员熟悉临床诊疗思维。
9. **最新临床指南查询模块的实现**
- **目标**: 提供面向基层的最新临床指南和专家共识，提供关键词快速查询。
10. **常见皮肤病图谱模块**
- **目标**: 提供基层常见皮肤病典型图文资料，便于快速比较和诊断。
11. **常见用药误区模块**
- **目标**: 提供常见用药误区的正确见解，避免用药方法不当而导致的治疗失败。
12. **同类药物的使用比较模块**
- **目标**: 向用户提供西医与祖国医学，是否含糖，性别，年龄等偏好设置。
13. **常用药物科普**
- **目标**: 向用户介绍明星药物的相关理论实践知识。
14. **响应式设计实现**
- **目标**: 确保应用在桌面端和移动端都有良好的用户体验。
15. **界面导航和用户体验**
- **目标**: 提供一个现代化、响应式的UI界面，提升用户在各种设备上的体验。
16. **药物配伍禁忌提示**
- **目标**: 对推荐药品组合进行配伍禁忌剔除。

### 技术架构
- **前端框架**: React.js
- **UI组件库**: 
  - Tailwind CSS 用于样式管理
  - Shadcn UI 提供现代化组件
  - Radix UI 提供无障碍性组件
- **构建工具**: Vite
- **移动端适配**: Capacitor用于Android平台构建
- **数据存储**: 静态JSON文件，无需后端服务器

### 主要功能模块
1. **诊断系统**
   - 症状选择与匹配
   - 诊断建议生成

2. **数据管理**

3. **用户界面**
